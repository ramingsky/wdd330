I like it that the book has a chapter 10 that talks about testing and debugging. You really need to know how to test and debug and its tools when you are into programming.

Errors are common and caused when something goes wrong in a program. It can be System Error, Programmer error and User error. Exceptions are a type of error that produces a return value and will give you information to help you deal with the error. Stack Traces are also produced by exceptions. They walk through the points that led to the exception. This helps you work backwards through the breadcrumbs that lead to the originating point of the error.

Warnings can be triggered if there's a small error, but the program keeps running (possibly incorrectly) despite the error. Different browsers throw out warnings differently, so you may or may not get one.

You can use strict mode to find more errors and warnings that may not be caught otherwise, helping you debug issues before they become big. It also catches deprecated and sloppy code. This page has been written using strict mode to practice. This is excellent to me, as I am also learning Java, which is strictly typed. My lazy Javascript habits have spilled into my Java coding, which throws errors at the tiniest thing, like a missing semicolon, which sloppy mode of Javascript accepts without hesitation. You can use strict mode globally (for all of the script), or locally within a single function. Modules use strict mode by default.

linting tools can be used to test the quality of JavaScript code. Linting tools are also useful for enforcing a programming style guide.

Handy also is feature detection to run particular code based on whether it is supported in the browser running the code, keeping that unsupported feature from crashing your program.

I much prefer breakpoints to alerts. Perhaps walking through each step of the code and watching the result of each command is a more effective method of debugging.

console.trace() and the debugger command are advance tools that can be handy and usually used most of the programmers.
