Chapter 8: CSS3 Transforms & Transitions

Transforms
  • Transform property:
    • lets you translate, rotate, scale or skew objects on a page
    • doesn't work on inline elements; use inline-block or just block to apply transforms
    • can be stacked, but shouldn't be repeated (second will replace first) ex: transform: translateX(10px) scale(.5);
  • Translation:
    • lets you move elements left, right, up, down, similar to position: relative -- using x/y axes.
    • does not impact the flow of the document
    • moves elements relative to their current position
    • transform: translate(x, y);
    • alternate option: transform: translateX(10px) or translateY(10px)
  • Scale:
    • transform: scale(x,y) -- if only x or y is provided, it will apply to both height and width. ex: scale(2) would double the size proportionally.
    • can also use scaleX(x) or scaleY(y) alone
    • does not impact document flow; embiggened stuff will overlap things around it.
  • Rotate:
    • rotates an item with a specified angle value
    • rotates items around the center point unless specified with transform-origin
    • positive degrees move clockwise, negative move counterclockwise
    • transform: rotate(10deg)
  • Skew:
    • specifies a skew along x/y axes by degrees
    • transform: skew(Xdeg, Ydeg);
  • Transform-origin property defaults to center; change for more dramatic results
  • Order matters! Transforms are applied in the order given; if you rotate first, the item will move on the rotated axis.
Transitions
  • Transitions apply when a class is changed; saves JavaScript some work.
    • Declare the original state of the element
    • Declare the altered state, ex. :hover
    • Include transition functions in the default/originating style with transition properties
  • transition-property: specifies which property of an element should show the transition happening; defaults to all
  • transition-duration: specifies how long animation should take, in s or ms. (200ms is optimum for basic transitions like hover effects)
  • transition-timing-function: controls the pace of the transition, ex: ease, linear, ease-in, ease-out, ease-in-out. Can also customize with cubic-bezier function. This is complex and beyond my desire to learn.
  • steps function -- a transition-timing function
    • define the number of steps and the direction of either start or end (where the first step happens)
    • steps(5, start) jumps through the animation equally over 5 steps starting with 0% and ending at 80%, whereas steps(5, end) would start with 20% and end with 100%.
  • transition-delay: how long to wait before starting the animation, in s or ms.
  • transition shorthand: declare property, duration, timing-function, delay, on one line.
  • Transitioning more than one thing at the same time in different ways: declare with commas and specify in order. Ex:
    • transition-property: transform, color (specifies two properties)
    • transition-duration: .2s, .1s (color will change faster than transform)
Animations
  • Allows you to define keyframes -- snapshots with the starting or ending point of the transition
  • Good alternative to JavaScript animation; better to use for simple-state changes especially on mobile
  • Keyframes:
    • Defined with @keyframes, then a name of your choosing (like a function declaration)
    • @keyframes myAnimation { keyframes here }
    • keyterms: from and to (0% and 100%)
    • Each keyframe includes properties to be animated.
  • animation-name: myAnimation; within an element's styles declares the animation (calls the keyframes function)
  • animation-duration (not strictly required but best practice)
  • animation-timing-function with same parameters as transition-timing-function above.
  • animation-direction: usually goes from 0-100% (normal), but can be specified to go backward with reverse or alternate forwards/backwards with alternate or alternate-reverse.
  • animation-delay: specifies how long to wait before starting the animation.
  • animation-fill-mode: specifies what happens before and after the animation (none by default).
    • forwards -- holds the last keyframe after animation ends
    • backwards -- loads the element with the first animated state and reverts to default after the animation ends
    • both -- will start out with the first animated state and hold onto the last animated state after it ends
  • animation-play-state: accepts running or paused; can be controlled with CSS or JS.
  • animation shorthand: name, duration, timing function, delay, iteration-count, direction, fill-mode, and play-state, declared in any order on one line.
  • "Just because you can doesn't mean you should." -- use transforms/animations with a light touch. Use only to enhance the user experience where it makes sense, and skip the extra flash.

Chapter 12: Canvas, SVG, and Drag and Drop

  • Canvas element allows user to draw on the page. It was developed by Apple, so familiarity with Canvas helps with some iOS developer skills (Quartz 2D).
  • Canvas tag -- text inside only shown if Canvas is not supported by the browser. Must specify width and height in HTML tag; CSS will not resize the Canvas coordinate system.
