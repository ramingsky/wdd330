Something that had me researching quite a bit was closures. The book was just not very clear to me. I first found an article titled Demystifying JavaScript Closures, Callbacks, and IIFEs that was very clear.

In JavaScript, a closure is any function that keeps reference to variables from its parent’s scope even after the parent has returned. This means practically any function can be considered a closure, because... a function can refer to, or have access to –
  • any variables and parameters in its own function scope
  • any variables and parameters of outer (parent) functions
  • any variables from the global scope.
So, chances are you’ve already used closures without even knowing it.
Essentially, the closure is a block within a block. You can refer to variables outside the closer. It stores a refence to its outer variable rather than a value.
