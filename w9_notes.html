Chapter 9: The Window Object

  • BOM = Browser Object Model (only applies to browsers, not other environments)
  • Global variables are created without const, let, or var. "window" is a global peroperty; all global variables in a browser are properties of "window."
  • parseInt() and isNaN() are global properties of window object.
  • ES6: parseInt() and isNaN() are also methods of Number object.
  • Dialogs such as alert(), confirm(), and prompt() are risky; can be blocked by browsers, or can cause errors by completely halting the program pending user input.
  • window.navigator.userAgent returns browser info, but can be unreliable. (deprecated)
  • window.location
    • .href returns entire URL (can also be designated by assignment, causing a redirect)
    • .protocol returns a string with the protocol used (http, https, pop2, ftp, etc
    • .host returns the domain name and port number, ex. www.sitepoint.com:44
    • .hostname returns the domain (no port number)
    • .port returns the port number (empty if not explicitly stated in URL)
    • .pathname returns the path AFTER the domain
    • .search returns a string starting with ? and query string parameters, if any
    • .hash returns a string starting with # and the fragment identifier if any.
    • .origin returns a string with the protocol and domain where the current page is -- cannot be changed by assignment (unlike href)
    • reload() method - forces a refresh of the page. Use reload(true) to load fresh from server (not cached)
    • assign('urlstring') method - loads another resource from a URL
    • replace() - similar to assign, but doesn't store current page in history (can't use Back button)
    • toString() - returns a string with the whole URL
  • window.history accesses previously-visited page info (not to be confused with HTML5 History API)
    • .length shows how many pages have been visited before this one
    • .go() can be used to go forward or back a page (0 reloads current page, positive/negative numbers go forward/back respectively)
    • .forward() goes forward one page
    • .back() goes back one page
  • window.open() - accepts three parameters: URL, window title, attributes list. Ex: const popup = window.open('https://sitepoint.com','SitePoint','width=400,height=44,resizable=yes');
  • popup.close() (where popup is a reference to a window) closes a window
  • window.moveTo() moves to coordinates given as x,y parameters
  • window.resizeTo() resizes; specify height and width
  • Note: Most popup methods are strongly discouraged nowadays.
  • window.screen.height and .width find the user's screen size; availHeight and availWidth give the screen size minus OS menus. Screen object is more useful in Mobile, but most of these are replaced by better practices now (like media queries/feature detection)
  • Document Object:
    • object within the window object
    • used extensively for selection: see DOM
    • document.write() replaces the current document with the string in parentheses -- if used inline, will only fill the space within script tags
  • Cookies: save a text file of name/value pairs with info from a user's previous session -- note EU Cookie Directive requires site to ask permission to use cookies.
  • To save a cookie: document.cookie = 'name=value; name=value;' etc
  • Adding more to a cookie will not overwrite the string; it will append new values at the end. Using the same name allows you to reassign values.
  • Read contents of the "cookie jar" with document.cookie
  • split() allows you to break the cookie into an array of name/value pairs, so you can use a loop to iterate through and use them
  • Cookies are session cookies by default (disappear after browser is closed) but can be set to expire later with "; expires=date" to the end of a cookie. (date must be in UTC string format, ex. expireDate.toUTCString() DO NOT rely on this for sensitive info.
  • Delete a cookie by setting the expiration date to in the past.
  • Timing Functions:
    • setTimeout() -- lets you set an arrow function and a number of milliseconds to count down before running next code. Can be cancelled with clearTimeout()
    • setInterval() repeatedly invokes callback function after a given number of millisecondsds

Chapter 14: HTML5 APIs

  • HTML 5.1 is the latest iteration of HTML, and includes several APIs that JS can access.
  • HTML5 was for HTML what ES6 was for JavaScript...
  • The data- attributes is a way of embedding data in a web page using custom attributes that are ignored by the browser... These attributes:
    • must start with data-
    • contain only lowercase letters, numbers, hyphens, dots, colons, underscores
    • include an optional string value
    • can be accessed with "dataset" ex: access data-powers with superman.dataset.powers
    • old browsers can get this with getAttribute, ex: superman.getAttribute('data-powers');
    • allow you to add data directly into the HTML markup for a richer US
    • should only be used when no other HTML elements apply (ex: data-class is stupid. Just use class attribute)
  • HTML5 APIs
    • The HTML5 dpecification contains a number of APIs that help to gain access to hardware, such as cameras, batteries, geolocation, and the graphic card.
  • Web Storage API: allows key-value pairs to be stored on the user's device, like cookies but different:
    • Not shared with the server on every request
    • Available in multiple browser windows if domain is the same
    • storage capacity is much larger (usually up to 5GB per domain -- cookie limit is 4KB)
    • Does not automatically expire like cookies (so cookies might be better for some things)
    • This is where localStorage comes from! Also sessionStorage, which is similar but only for the current session.
  • localStorage/sessionStorage methods:
    • setItem('name','value') - stores the information
    • localStorage.name = 'value' - stores the information (by assignment)
    • getItem('name') - returns the value stored with that name
    • removeItem('name') - removes the item
    • delete localStorage.name - deletes the item
    • localStorage.clear(); - completely removes everything in LS
    • LS methods fire a storage event -- this has properties of its own: key, newValue, oldValue, and storageArea (the latter tells us where the changed item was saved)
  • LocalStorage only accepts strings; use JSON.stringify() to save objects as strings
  • Geolocation API: lets you access the geographic coordinates of the user's device (with user's permission). If available, uses "navigator" object and methods:
    • navigator.geolocation.getCurrentPosition(callback); - passes a "position" object with coords as latitude and longitude. ex: position.coords.latitude, position.coords.longitude
    • position.speed - ground speed of device in meters per second
    • position.altitude - estimate of device's altitude in meters (above center of Earth)
    • position.heading - direction the device is moving in
    • position.timestamp - time that position info was recorded
    • position.accuracy - returns accuracy of the coordinates in meters (lower = more accurate)
    • navigator.geolocation.watchPosition(callback) calls the callback function every time the device's position updates (stop this happening with .clearWatch(id).
  • Web Worker API: performs computationally intensive tasks in the background (helps websites run more smoothly)
  • Websockets: new protocol for internet communication; allows real-time, two way communication
  • Notification API: allows you to display notifications on the user's system
    • Get permission first. if(window.Notification){ Notification.requestPermission();}
    • .then((permission => if(Notification.permission === 'granted') { new Notification('string');
    • Notification object includes body and icon you can specify as strings. Ex: const notification = new Notification('JavaScript:Novice to Ninja',{body: 'The new book from Sitepoint', icon: 'sitepointlogo.png'});
    • Close a notification with notification,close();
  • audio and video elements: embeds audio and video in a webpage; Media API helps control these via JS
  • canvas element: dynamically draws geometric shapes, text, and images in real time with JS
  • shim/polyfill: a piece of code that adds support of missing features to older browsers.
